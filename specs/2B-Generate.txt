### LaTeX Escaping Rule

All LaTeX within JavaScript string values must use **double backslashes**.
Single backslashes cause JSON/JS parse errors.

| Correct | Incorrect |
|---|---|
| `"\\frac{1}{2}"` | `"\frac{1}{2}"` |
| `"a^2 + b^2 = c^2"` | fine — no backslash needed |
| `"\\sqrt{25} = 5"` | `"\sqrt{25} = 5"` |

### uid Format

Use `makeUID(type, level, diff)` which returns a string in this format:

```
{type_abbr}-{NNN}-lev{L}-d{D}
```

| type | abbr |
|---|---|
| NUMERIC | num |
| TEXT | txt |
| MCQ | mcq |
| MATCH | mat |
| SPOT_ERROR/STEP | sep |
| SPOT_ERROR/VALUE | sev |
| EXPLANATION | exp |

Example: `makeUID("NUMERIC", 3, 2)` → `"num-007-lev3-d2"`

The counter NNN increments across all questions in the module.

## QUESTION GENERATION RULES

**Pool-and-Remove Pattern:**
For each level, maintain a pool of context scenarios (e.g. object names, settings,
narratives) as listed in the Review Document Dev notes. `pickAndRemove()` selects from
the pool so no context repeats within a session. When the pool is exhausted, reset it
silently (refill from the original array).

**Type Mix:**
Implement the type proportions as specified in the Review Document. `getQuestion(level,
diff)` must dispatch to the correct generator function for the given level and diff,
consistent with the mix stated in the Review Document.

**Difficulty Distribution:**
Implement the difficulty spread across `questions_per_level` as specified in the Review
Document. Do not generate all questions at the same difficulty.

**Skill Fidelity:**
Each level's generator functions must test only the skill defined for that level in the
Review Document. Do not mix skills across levels.

**SVG Diagrams (if include_svg is Yes in the Review Document):**
Generate diagrams as inline `data:image/svg+xml` URI strings. Set `q.img` to the URI
and `q.imgAlt` to the accessibility description stated in the Review Document. When
`q.img === ""`, `renderFront()` must apply a CSS class that centres the question text
both horizontally and vertically.

Never pass algebraic variable names directly to SVG coordinate slots. Always map the
question's computed values to the correct geometric position explicitly.

**MCQ:**
All four MCQ options must be computed values derived from the question parameters,
matching the distractor table in the Review Document. Placeholder strings such as
"incorrect option A" are never acceptable.

**File Size Management:**
The output module will exceed the Write tool's size limit. Always write the file in two
stages: use the Write tool for the main body up to and including `generateSolution`, then
use a Python bash script to append `referenceItems`, `referenceLabel`, the closing brace
of the config object, and the boot line. Verify completeness after writing by checking
that the boot line `QsetFW.init(...)` is present as the final non-blank line.
